Responsive Redesign

I chose this website because it really stuck out to me how simple it was/how little work seemingly went into it. It literally looks like a document created in Microsoft Word, and for a company with 327,000 employees and almost $1 trillion worth of assets under its control, this was surprising… it obviously doesn’t hurt their business. The website is very consistent with the values, philosophy, and guiding principles of the firm. The simplicity is intentional, it is meant to inform users at a very basic level with no bells or whistles. Their performance speaks for itself. You can check out the website for yourself at https://www.berkshirehathaway.com/


Interface Problems

1. You are brought to separate subpages when you click some of the bulleted links in the table, which could have been avoided by using sections on the main page or expanding dropdown menus under the link that show you more links.
2. It is difficult to learn/figure out where you are supposed to click/what elements to interact with before reading the whole link first.
3. There is no structure or grouping to learn, you have to read the full text of each item in an unordered list, it would be pretty unrealistic to memorize a certain link at a certain spot.
4. The subtext that most often includes when the last edited date was is very small and difficult to read, and on mobile, I have to zoom in to read it.
5. There isn’t a real about page or info on contacting the company via the website, there is one line saying you can write to the corporate office for comments.
6. The website is purely text and could have been written in Microsoft Word, there isn’t any dynamic content.
7. There is not much consistency in the links in the list, some of them navigate to external sites, and some bring you to another page on the website with more links.
8. The user has to read the small subtitled text below links to find out if there are any updates and when the latest update to each page was.
9. The website doesn’t have an interface that makes it easier for a user to get to where they need to go, it just puts everything they could ask for in (almost) one place and leaves the rest to them.

Accessibility

Using WebAIM WAVE to find possible accessibility problems, there were a few problems detected. I agree with some of the problems detected by the tool, with the main one being the small and hard-to-read text underneath each item in the table which contains an unordered list of bulleted hyperlinks. The tool also detected that the site did not have any heading structure or page regions, but I didn’t see that as too big of an issue as there is a header with the company name and address, as well as horizontal lines separating the different sections of the website.
